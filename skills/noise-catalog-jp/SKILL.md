# 日本語テキスト検索のノイズカタログ

分かち書きのない日本語コーパスで、キーワード検索・意味検索を設計するときの既知の罠と対策。
出典: 国会会議録（約80年分）×キーワード収集の実測（DEPT.405「友人・政治まさはる」2026-07）。すべて実例つき。

## ノイズ型一覧

### 型1: 多義語型

- **実例**: 「姿勢」で検索→5,442件。中身はほぼ「政府の姿勢」「政治姿勢」で、身体の姿勢は僅少。
- **兆候**: 件数が同カテゴリの他の語より一桁以上多い。全期間比が平坦（特定の時代に集中しない）。
- **対策**: 件数だけで判断せずサンプルを読む（型検出の基本動作）。応急=共起語で絞る複合検索。本命=埋め込みベースの意味フィルタ。

### 型2: 部分一致型（日本語固有）

- **実例（一般化）**: 2文字程度の短い検索語が、無関係な複合語の内部（末尾・中間）にたまたま一致しただけの結果で大半（実測で4割〜8割程度）を占めたケースがあった。
- **原因**: 分かち書きがない言語では、文字列検索が単語境界を知らない。より長い語の内部にターゲット語が埋まる。
- **兆候**: ヒット文書の主題が突然まったく別分野になる。
- **対策**: 応急=既知の包含語（衝突する具体的な複合語）を除外リスト化する。中期=形態素解析による単語境界チェックの後段フィルタ。本命=意味フィルタ。除外語はカタログ化して再利用する。

### 型3: 定型文・議事進行混入型

- **実例**: 議事録の「○○君」（議長の指名発言）、「ただいま議題となりました〜」（趣旨説明の定型冒頭）、決算報告の定型文がサンプルに混入し、クラスタや評価対象を汚す。
- **兆候**: 極端に短い文書、同一パターンの繰り返し、人間の採点で「判定材料不足」の棄権が集中。
- **対策**: メタデータで除外（話者役割=議事進行、文書種別）。メタデータが無ければ文長閾値＋定型パターンマッチ。**人間の「なんか変」コメントを必ず起票する**——検品コメントは対策の一次情報。

### 型4: 表記正規化型

- **実例**: 国会会議録では英数字が全角格納されることがある（「ＩＴ」）。半角「IT」のLIKE検索では取りこぼす。
- **対策**: NFKC正規化した派生列（例: speech_norm）を作り、検索・埋め込みは正規化列を基準にする。**原文列は非破壊で保持**（正本主義）。カタカナ・ひらがな・旧字体のゆれも同じ枠で扱う。

### 型5: 語彙の時代変遷型

- **実例**: 行政語彙は年代で変わる——「看護婦」→「看護師」（2002年に法改正で名称変更）。「看護師」だけで検索すると2002年以前の同一議題を取りこぼす。
- **対策**: 検索語（特に制度語・行政語）には**有効年代**を付与し、期間に応じて新旧語彙を切り替え・併用する。時代をまたぐ収集では変遷表を先に作る。

## 検出プロトコル

1. **件数だけで計画しない。** プロファイリング（件数調査）の後、必ずサンプリングEDA（数十件→クラスタ→ラベル）を挟む。
2. **クラスタラベルは誤爆を自己申告する。** 意味の地図（クラスタ＋ラベル）を作ると、汚染クラスタは「〜の誤爆」「別文脈」という名札で浮かび上がる——EDAは検品装置を兼ねる。
3. **人間のコメントは要件文。** 採点・レビュー中の「文脈で区別してほしい」「これはセルが違う気がする」は、そのまま次工程のフィルタ要件として起票する。
4. **新種を踏んだらカタログに追記する。** このカタログ自体を追記型で運用する（ノイズ型は資産）。

## 対策の階層

- **応急**（当日）: SQL/フィルタでの除外条件、文長閾値、正規化列。
- **本命**（次版）: 埋め込みベースの意味フィルタで「語の一致≠主題の一致」を吸収。
- **恒久**（文化）: 除外語・変遷表・ノイズ型をリポジトリのカタログ文書として蓄積し、新規プロジェクトの設計初期に参照する。
