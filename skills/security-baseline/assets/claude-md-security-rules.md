# CLAUDE.md 転記用: セキュリティ共通指示テンプレート

新規プロジェクト立ち上げ時、以下のブロックをそのままプロジェクトの CLAUDE.md
（spec-driven-dev の「絶対要件」節）へ貼る。実装指示の差はこうなる——
Before:「ログイン機能を実装して」→ After:「ログイン機能を実装。下記の必須条件をすべて守ること」。

---

## セキュリティ必須条件（全実装に適用）

1. 認可チェックは必ずサーバ側で実施する（フロントから渡された ID・role を信頼しない。
   リソース取得はログインユーザーの所有者照合を DB 条件に含める）
2. SQL は ORM またはプリペアドステートメントを使用する（文字列連結 SQL 禁止）
3. シークレット・API キーは環境変数から取得する（コードに直書き禁止・クライアント配布物に含めない）
4. エラーメッセージは内部情報を含めない（スタックトレース表示禁止。例外は握り潰さず記録し、
   fail-closed に倒す）
5. 使用ライブラリは最新安定版を指定し、バージョンを固定する（lockfile をコミット）
6. セッション・トークンの有効期限を設計段階で決める（サーバ側で署名・期限・失効を検証）
7. 入力値は必ずサーバ側で検証・エスケープする

上記に反する実装が必要に見えた場合は、実装せずに止まって報告すること。
